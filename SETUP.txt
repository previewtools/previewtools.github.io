PROJECT STRUCTURE:
==================

Root Directory:
├── index.html              (Main homepage)
├── tools.html              (All tools page)
├── manifest.json           (PWA manifest)
├── sitemap.xml             (SEO sitemap)
├── robots.txt              (Robots.txt for SEO)
├── css/
│   ├── styles.css          (Global styles & animations)
│   └── pages.css           (Page-specific styles)
├── js/
│   ├── component-loader.js (Loads reusable components)
│   └── slider.js           (Hero slider functionality)
├── components/
│   ├── header.html         (Reusable header)
│   ├── footer.html         (Reusable footer with widgets)
│   ├── sidebar.html        (Category sidebar navigation)
│   └── all-tools.html      (All tools grid component)
└── images/
    ├── logo.jpg            (Brand logo)
    └── favicon.jpg         (Favicon icon)


DEPLOYMENT:
===========

1. For GitHub Pages (https://previewtools.github.io/):
   - Push all files to your gh-pages branch
   - Enable GitHub Pages in repository settings
   - Point domain to GitHub Pages in DNS settings

2. For Local Development:
   - Use any simple HTTP server
   - Python: python -m http.server 8000
   - Node.js: npx http-server
   - PHP: php -S localhost:8000

CUSTOMIZATION TIPS:
===================

1. Colors: Edit CSS variables in styles.css
   - Change primary colors in :root {}
   - Update gradients in component styles

2. Content: Edit HTML directly
   - Add/remove tool cards in all-tools.html
   - Update descriptions and links
   - Modify feature list and use cases

3. Images: Replace with your own
   - logo.jpg (brand logo)
   - favicon.jpg (icon)
   - Update image paths in components

4. Domain: Change all URLs
   - Update OG tags with your domain
   - Update sitemap URLs
   - Update manifest.json start_url


COMPONENTS USAGE:
=================

Components are loaded dynamically via component-loader.js

To add a new component:
1. Create HTML file in components/ folder
2. Add entry to ComponentLoader.components object
3. Add container div with matching ID
4. Components load on page load automatically
